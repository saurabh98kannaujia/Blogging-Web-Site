What is Nginx ?

Devops Published at: 26 January 2024

In this blog we will learn about nginx, the internal architecture of nginx and the installation of nginx with nginx commands:

Welcome to the world of Nginx, a powerhouse in the realm of web servers and reverse proxies. In this blog, we' will learn the capabilities of Nginx, exploring how it can boost web infrastructure for optimal performance, scalability, and reliability.

What is Nginx ?

  • NGINX is open source software for web services, backup, caching, load balancing, media streaming and more.
  • It is a web server originally designed for maximum performance and stability.
  • In addition to HTTP server features, NGINX can be a proxy server for email (IMAP, POP3, and SMTP), as well as reverse proxy and load balancing for HTTP, TCP, and UDP servers.

Nginx Architecture:

NGINX follows master-slave architecture and supports event-driven, asynchronous and non-blocking models.
NGINX is divided into three different parts. These are
  • Master
  • Worker
  • Cache

Master:

  • In the master-slave architecture of NGINX, tasks are assigned to workers based on client requests.
  • The master efficiently delegates jobs to workers and immediately proceeds to handle the next client request without waiting for worker responses.
  • Once workers complete their tasks, the master promptly forwards the responses to the respective clients.

Worker:

  • In NGINX architecture, workers act as slave and following the instructions of the master.
  • Each worker is capable of efficiently managing more then 1000 requests concurrently, employing a single-threaded approach.
  • After completing the assigned tasks, the worker communicates the response back to the master.
  • This single-threaded model is advantageous as it conserves RAM and ROM space by operating within the same memory space, in contrast to the multi-threaded approach, which functions across distinct memory spaces.

Cache:

  • Nginx cache speeds up the page rendering by retrieving content from cache memory instead of fetching it from the server.
  • The cache stores pages upon the initial request, allowing subsequent requests for the same page to be quickly served from the cache.

Features:

  • High Performance
  • Web Server
  • Reverse Proxy
  • Load Balancing
  • SSL/TLS Termination
  • Caching Strategies
  • Asynchronous and Event-Driven
  • Scalability

Steps to install nginx on Linux:

Below are the installation steps:
1: Update Package Lists:
sudo apt update
2: Install NGINX:
sudo apt install nginx
3: Start NGINX:
sudo systemctl start nginx or sudo service nginx start
4: Enable NGINX to Start on Boot:
sudo systemctl enable nginx
5: Restart NGINX Service:
sudo systemctl restart nginx or sudo service nginx restart
6: Stop NGINX Service:
sudo systemctl stop nginx or sudo service nginx stop
7: Check status of NGINX:
sudo systemctl status nginx or sudo service nginx status
