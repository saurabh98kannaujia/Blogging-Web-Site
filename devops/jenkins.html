What is Jenkins ?

What is Jenkins and How it is work ?

Jenkins CI/CD pipeline is a crucial component in the realm of DevOps, where CI stands for Continuous Integration and CD encompasses Continuous Deployment.

Before Jenkins first lets understand about what is CI/CD?

  • Continuous Integration and Continuous Deployment are vital in the world of DevOps tools, streamlining and automating the various stages of software development and delivery.
  • This process involves a series of steps, organized into a pipeline, that are executed during the software product delivery lifecycle. These steps typically include activities such as building the code, conducting tests, and deploying the application.

Why is CICD important ?

  • The significance of CI/CD lies in its ability to automate processes, minimizing the potential for human error and ensuring a consistent approach to software deployment.
  • The CI/CD pipeline facilitates the automation of tasks such as code compilation, execution of unit tests, code analysis, and security checks. This automation not only expedites the delivery process but also enhances the overall reliability and quality of the software being developed and deployed.

What is Jenkins

  • Jenkins is a Java-based open-source automation platform that serves as an automation server for facilitating continuous integration and continuous delivery (CI/CD) in software development.
  • Its core functionality revolves around automating key stages of the development process, including build, test, and deployment.
  • Functioning as a crucial tool for continuous integration, Jenkins is equipped with a variety of plugins tailored to support various aspects of the CI/CD pipeline.
  • This platform plays a pivotal role in the seamless creation and testing of software projects, providing developers and DevOps engineers with a streamlined approach to integrating changes into projects.
  • Additionally, Jenkins simplifies the process for end-users to access and deploy new builds. Its versatility extends to supporting continuous release practices by integrating with diverse testing and deployment methods.

What is Jenkins Pipeline ?

  • A Jenkins pipeline is like a to-do list for Jenkins.
  • It's a set of steps that Jenkins follows to get things done in the software development process.
  • Imagine it as a checklist of tasks, where each task is connected to the next one. This checklist, or pipeline, is made possible by using plugins in Jenkins, making it easy to create and manage the steps needed for continuous integration and delivery.

Steps to Create a pipeline:

When you install Jenkins on the machine, it is running on an 8080 port.
Step 1: open Jenkins web interface on browser using http://localhost:8080
Step 2: Click on the Create a Job or New Item and you get the below screen
Step 3: In the item name field enter your pipeline Name here i take a pipeline as test and there are multiple types of pipeline based on the requirement. So here we are taking Pipeline and click Ok.
Step 4: After clicking on OK you get the configuration screen directly. There is a three section in Configuration:
  1. General
  2. Advance Project Options
  3. Pipeline
Step 5: Click on Advanced Project Options or you can scroll down then you get the pipeline section. In Pipeline section there is 2 options:
  1. Pipeline Script
  2. Pipeline Script From SCM (from github)
So we are going to follow Pipeline Script Method
Step 6: In the Script we have write the script
pipeline { agent any stages { stage('Installing dependencies') { steps { echo 'Fetch latest code' } } stage('Build') { steps { echo 'Build Started' } } stage('Publish') { steps { echo 'Push the code on Repo' } } stage('Deploy') { steps { echo 'Deploy the build on server' } } } }
After writing the script click on Save button.
Step 7: After saving the script click on Build Now.
Step 8: After clicking on build now all the stages of the pipeline will be executed.
